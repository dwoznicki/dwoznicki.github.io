Ruby Classes

This week, we spent a fair amount of time studying classes in Ruby. Classes are very important in Ruby as they define the characteristics of objects (such as strings, integers, etc.) as well as what methods you can use with them. For example, the string class tells the computer how to handle a string, as well as that the #length method should return an integer equal to the number of characters inside the string. But classes aren't limited to pre existing ones; you can make your own as well. But how would you make a class? What goes into it? Well, we're going to look at a sample class today.

Our Sample Class

A good way to start is to imagine your class as a real world object. In this case, we're going to make a Dog class.

Let's start out by defining our dog class. 


      > class Dog
      >
      > end

Now we need to consider what attributes and actions our Dog class will have. Classes excel when you want to making several instances of something, but all with the same basic characteristics. Not all dogs are the same, but they do all have a lot in common. So what makes a Dog a Dog?

Instance Variables

Picture a dog in your mind. Good. Now picture another. And another. Okay, stop picturing dogs (if you want). What qualities did your dogs possess? What differentiated each one from the others? In Ruby, we can define these qualities inside a class using something called instance variables. Instance variables are common attributes that you can assign and change for each instance of the class. One way to define these attributes is with the Ruby class method initialize. For example: 


      > Class Dog
      >   def initialize(name, age, breed, bark)
      >     @name = name
      >     @age = age
      >     @breed = breed
      >     @bark = bark
      >   end
      > end

This class has four instance variables: @name, @age, @breed, and @bark. When the user creates a new Dog, they can set these instance variables by passing in the three arguments they want to use for their instance of the class. In this case, we'll use this guy: 


      > howard = Dog.new("Howard", 2, "Old English Sheepdog", "Woof woof!")
      =>Dog: @name="Howard", @age=2, @breed="Old English Sheepdog", @bark="Woof woof!"

Notice anything familiar? Making a new instance of the Dog class uses the same syntax as you would for making a new array, or a new hash. The ClassName.new method is common in Ruby when making a new object, and your classes are no different.

So what now? We have our dog, what can it do?

Instance Methods

If instance variables define what a Dog is, then instance methods define what a Dog can do. These are the actions your dog can take, and those you can take upon it. This can get overwhelming very quickly, so let's consider only one action for our Dog for now: barking. We can define this method inside our class much the same way you'd define any other method: 


      > Class Dog
      >   def initialize(name, age, breed)
      >     @name = name
      >     @age = age
      >     @breed = breed
      >     @bark = bark
      >   end
      >
      >   def sound
      >     puts bark
      >   end
      > end

Now, when we make a new dog, we can call the sound method to hear its bark. 


      > howard = Dog.new("Howard", 2, "Old English Sheepdog", "Woof woof!")
      =>Dog: @name="Howard", @age=2, @breed="Old English Sheepdog", @bark="Woof woof!"
      > howard.sound
      => "Woof woof!"

Again, we call the method on our class the same way you'd call a method on any other class. Instance methods allow for a lot of flexibility and power in classes. These are what set classes apart and make them extremely useful tools in coding.
