Ruby vs. JavaScript

This week our class started looking into JavaScript. JavaScript (JS) is a powerful language that adds a lot of functionality to web apps. It has some very simple integration with HTML and is widely supported across browsers, making it ideal for adding script to a web page.

That said, it's certainly been a challenge switching over from Ruby to JS for me. Ruby's high level, clean look is hard to maintain in JS (for me at least). I can't help but look back fondly and compare some of the concepts I'm studying in JS with things I learned in Ruby. So in this post, I'm going to compare two important objects: the Ruby class and the JavaScript constructor function.

What is a Ruby class?

The Ruby class is an object that can be used as a guideline for making other objects. It's a blueprint of sorts. Classes can inherit properties and methods from their parents, and indeed often do. Ruby is a language of classes, in fact. Objects are just instances of classes, which is why you can always use the new method to make new strings, arrays, etc.

The syntax for a Ruby class looks like this: 


      > class Example
      >   def initialize
      >     @sentence = "This is an example sentence."
      >   end
      >   def talk
      >     puts @sentence
      >   end
      > end
      >
      > instance = Example.new
      > instance.talk
      => This is an example class.

There are a few things to note here. First, the initialize method is the first thing that runs when you make a new instance of your class. This sets up all the properties for the class. Second, you can instantiate the class using the new method. New is a method of the class object in Ruby. Lastly, you can define instance variable inside the class. Instance variables are marked by the @ symbol before the name. These variables can be used by any method inside the class; in this case, talk.

What is a JavaScript constructor?

Unlike the Ruby class, the JS constructor is actually a function. This was a little odd to me at first since I associated functions with Ruby methods. But the JS constructor function actually plays a very similar role to the Ruby class. Let's look at an example: 


      > function Example() {
      >   this.sentence = "This is also an example sentence.";
      >   this.talk = function() {
      >     console.log(this.sentence);
      >   }
      > }
      >
      > var instance = new Example();
      > instance.talk();
      => This is also an example sentence.

It's pretty similar, but not quite the same. The first thing you might notice is that there's no method to initialize the constructor. Since the constructor is a function, we can simply assign the properties (this.sentence) at the beginning and that code will run first. This binds the properties to the constructor function, similar to a Ruby instance variable or method. However, the most important piece is probably the new operator.

New does a number of things.

  1. makes a new object
  2. connects the new object to the constructor
  3. binds this to new object
  4. runs the constructor function

This is what really defines an instance of the constructor function, and makes it similar to a Ruby class.

That's all for this week. Thanks for reading.
