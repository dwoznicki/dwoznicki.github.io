Display: inline vs. block vs. inline-block

This week we started looking into CSS (Cascading Style Sheets) to format the HTML practice we did last week. CSS is an integral part of web design. All the colors, fonts, buttons, menus, etc. that we have come to expect from websites are made in CSS. But for such an important part of the web, it sure can be abstruse to work with. Understanding all the various properties, what they do, and how they interact with each other is no mean feat, and I certainly don't mean to tackle it all in this post. So I'm going to focus on one property here: display.

What is display?

The display property indicates how an element is going to interact with other elements on the page. Is it going to take up a set amount of space? Is it going to sit alone on its line? There are three main properties to choose from when setting display: inline, block, and inline-block.

Display: inline;

An element displayed inline will have 3 basic attributes.

  1. It will respect margins and padding on the left/right, but not the top/bottom.
  2. It cannot have a set height or width. It will only take up as much space as necessary.
  3. It will allow other elements to sit on the same line as it.

Inline is best for text or elements you want to appear in a paragraph. It's not good for pictures since it doesn't have a height or width.Here's an example of what inline elements look like.

Display: block;

A block element is pretty much what it sounds like. It's a block of space on the page. Here are its attributes.

  1. It will respect all margins and padding.
  2. It can, and generally does, have a width and height.
  3. It will force a line break after the element.

Blocks are useful for large pictures or spaces that should take up a whole line themselves. Here's an example of how blocks look.

Display: inline-block;

The last display value, inline-block, takes some attributes from both of its brethren. Here's how it works.

  1. It will respect all margins and padding.
  2. It often has a height and width.
  3. It will allow other elements to sit on the same line as it.

Inline-blocks are ideal for pictures or spaces that need to take up space, but don't necessarily need a whole line. They're good for navigation menus and pictures in paragraph. Here's an example.
