The Enumerable#map Method

This week we continued to learn about ruby. One of our focuses was ruby Enumerables and how to interact with them. To that end, I'll be talking about a very useful method ruby has built in for working with Enumerables: map. But let's start with Enumerables.

What is an Enumerable?

Enumerables fall into an odd ruby category called "modules". Modules are containers for methods and constants that can be applied to multiple unrelated classes. Normally, you could only share methods through inheritance. But ruby gives you the option of writing methods that work with several objects without them needing to be related to one another.

This is all very handy for Enumerables which apply to collections of objects in ruby. Some common collections you'll see are:

  • Arrays [1, 2, 3]
  • Hashes { name: "Ken", type: "leopard gecko", age: 4 }
  • Ranges (1..10)

Although all these objects do not fall under a single superclass, ruby still has a list of methods, Enumerables, that can by applied to them. One of these commands is map. Let's look at it now.

What is map and how does it work?

Map is one of ruby's built in Enumerable methods that you can apply to collections, such as the ones above. Map will take a collection and apply a block of code for each object in the collection, returning an array of the results. Sounds a bit like each right? It should because all methods in the Enumerable module are based on iterating over a collection. What makes map special is that it returns an array after its done. The syntax looks like this: 


      collection.map { |object| block }
      => [array]

If you don't pass in a block, map will return an Enumerator with the original collection. From there, you're free to make it do what you please.

How can I use map?

Map is perfect when you want to output an array. Instead of going through the process of making a new array, iterating over the collection and pushing each object into the new array, you can simply use map! So let's say you had a range of numbers, like so:

range = 1..10

Your goal is to take every number in range and square it, putting the squares in a new array squared. This is the perfect kind of situation for map. 


      >> range = 1..10
      >> squared = range.map { |num| num**2 }
      =>[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

Note, map will apply any block of code for each item in the collection. Although it might be odd, the block doesn't necessarily need to be related to the collection. That means 


      >> ["a", "b", "c"].map {"chicken"}
      => ["chicken", "chicken", "chicken"]

is totally valid as far as ruby is concerned. All its interested in doing is applying the block once for each item in the collection.
